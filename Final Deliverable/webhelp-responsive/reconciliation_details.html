Reconciliation Details

Overview: Reconciliation Details report enables the CDI Manager to analyze the details around the reconciled encounters and identify gaps in the reconciliation process.

The report provides answers to the following questions.
  • Which CDSs are not following the reconciliation best practices?
  • Which CDS could benefit from additional education on the Reconciliation process?
  • How do we find the gaps in the reconciliation process?
  • Which cases were incorrectly reconciled?
  • What is the reimbursement impact the CDS had on a case?

Navigation: On the Analytics home page left side panel, navigate to CDE One > Executive Summary > Reconciliation Details.

Reconciliation Details displays the following:
  • KPI Cards: Total Reconciled and Overall Reimbursement Impact
  • Encounter Details Tabular Chart

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets generated in the tabular chart.

For filters on all pages, refer Filters on all Pages.

For information about footnotes, refer Footnotes.

Variance

The shift in the WT at the encounter level.

Example: 1.0178

Encounter Variance = Final Coded DRG WT - Reconciliation Assessment DRG WT (or Variance = Final Coded DRG WT - Working DRG WT when Discharge Date < 11/14/19 or Reconciliation Assessment DRG is blank)

Note:

Encounter Variance can be negative, positive, or zero.

Negative ‘Variance’ is displayed in the red font E.g. - 1.0178.

Encounter Variance is not calculated for the encounters that do not have an initial review date.

Review Not Needed Encounters are not considered.

Rounded to 4 decimal places.

Blended Rate

The Blended Rate associated with the payer/Grouper assigned to the encounter.

Example: 7512

When the Blended Rate is not available, the Operating Federal rate is displayed.
Reimbursement Impact

The ‘Reimbursement Impact’ value for the encounter.

Example: $ 5,234

'Reimbursement Impact' for an encounter is calculated using the below formula:

  • Encounter Reimbursement Impact = Encounter Variance x Blended Rate
  • Rounded to 0 decimal places.
  • Negative ‘Reimbursement Impact ($)’ is displayed in the red font.

Table 3. Report Slicers
Slicer Description
Discharge Date
  • Display the discharge date range.
  • You can select date from the lookup provided.
  • Example: 7/1/2020 - 7/31/2020
  • Default: Last 3 calendar months date range selected by default.
Reset
  • With the Reset button, you can reset the discharge date range.
  • Tooltip: Use "Ctrl + Click" to reset the Discharge Date.
Facility
  • Displays both active and inactive facilities in alphabetical order that you have access to.
  • You can select multiple facilities.
  • You can search the facility by Facility ID or Description.
  • Format: Facility Description (Facility ID)
  • Default: ALL
Payer
  • Displays both active and inactive payers in alphabetical order associated with the facilities you have access to.
  • You can select multiple payers.
  • "Unassigned Payer" displays data for the encounters that do not have a payer assigned.
  • You can search the Payer by Payer ID or Description.
  • Format: Payer Description (Payer ID)
  • Default: ALL
Grouper Type

Displays a list of available Grouper Type - - CMS, APR, LTCH, IPF, TRICARE

Default: CMS is selected

You can select single or multiple Grouper Type.

You can search the required Grouper Type.
DRG type

Displays the list of DRG type (Medical, Surgical, Unknown) associated with the Final DRG Code.

Values:

  • S
  • M
  • U

Default: ALL

You can select single or all DRG Type.
Reimbursement Impact
  • Displays the following values:
    • Positive Impact
    • Negative Impact
    • No Impact
  • By default, Positive Impact and No Impact are selected.

Positive Impact: Allows you to filter the list of encounters that have a positive reimbursement impact.

Negative Impact: Allows you to filter the list of encounters that have a negative reimbursement impact.

No Impact: Allows you to filter the list of encounters that have $0 reimbursement impact.
Impact Type
  • Displays the following values:
    • Financial Impact
    • Both Impact
    • SOI/ROM Impact
    • No Impact
    • Unassigned Impact
    • Default: ALL
