Encounter Details

Overview: Encounter Details report helps CDI Manager to analyze the details around the encounters and work with the CDSs to take appropriate action.

The report provides answers to the following questions:
  • Which CDSs are not following the reconciliation best practices?
  • Which CDS could benefit from additional education on the Reconciliation process?
  • How do we find the gaps in the reconciliation process?
  • Which cases were incorrectly reconciled?
  • What is the reimbursement impact the CDS had on a case?

Navigation: On the Analytics home page left side panel, navigate to CDE One > CDI Operations > Encounter Details.

Encounter Details displays the following:
  • KPI Card: Total Encounters
  • Encounter and Review Details

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets generated in the table.

For filters all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Variance

The shift in the WT at the encounter level.

Note: Variance can be negative, positive, or zero. Negative ‘Variance’ is displayed in red font E.g. - 1.0178.

Rounded to 4 decimal places. Example: 1.0178

Variance = Final Coded DRG WT - Reconciliation Assessment DRG WT (or Variance = Final Coded DRG WT - Working DRG WT when Discharge Date < 11/14/19 or Reconciliation Assessment DRG is blank)

For encounters with blank Final DRG WT, Variance = Possible DRG WT - Working DRG WT ( If Possible DRG/WT is blank, Working DRG WT is considered. In this case Variance will always br zero).

Variance is not calculated for the encounters that do not have an initial review date.

Review Not Needed Encounters are not considered.

Blended Rate

The Blended Rate associated with the payer/Grouper assigned to the encounter.

When the Blended Rate is not available, the Operating Federal rate is displayed.

Example: $ 5,234
Reimbursement Impact

The ‘Reimbursement Impact’ value for the encounter.

  • Rounded to 0 decimal places. Example: $ 5,234
  • Negative ‘Reimbursement Impact ($)’ is displayed in the red font.
 Reimbursement Impact = Variance x Blended Rate

Table 3. Report Slicers
Slicer Description
Admit Date

  • Example: 7/1/2020 - 7/31/2020
  • Default: Last 1 calendar months date range selected by default.
Discharge Date
  • Displays the discharge date range here. You can select required dates from the date lookup.

  • Example: 7/1/2020 - 7/31/2020
Reset
  • With the Reset button, you can reset the admit date range.
  • Tooltip: Use "Ctrl + Click" to reset the both Admit Date and Discharge Date.
Facility
  • Displays a list of active and inactive facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple facilities
  • You can search the facility by Facility ID or Description.
  • Format: Facility Description (Facility ID)
  • Default: ALL
Payer
  • Displays a list of active and inactive payers associated with the facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple payers
  • "Unassigned Payer" displays data for the encounters that do not have a payer assigned.
  • You can search the Payer by Payer ID or Description.
  • Format: Payer Description (Payer ID)
  • Default: ALL
CDS
  • Displays a list of active and inactive CDSs associated with the facilities you have access to.
  • The CDS Filter in all the reports displays the users with at-least CDS or CDS Manager role assigned.
  • "Unassigned CDS" displays data for the encounters that do not have a CDS assigned.
  • Format: Last Name, First Name MI (CDS ID)
  • Default: ALL
