Encounter Monitoring

Overview: Encounter Monitoring report enables the CDI Manager to manage the daily workload of the CDS team by analyzing unassigned encounters, concurrent encounters, not-reconciled and all encounters.

The report provides answers to the following business questions.
  • What does encounter distribution by status, unit, or payer look like? Does any status look like it’s becoming a problem? Is the team falling behind with reconciliation?
  • How many cases are unassigned? If yes, do any of my CDSs have the capacity to pick these cases up?
  • If cases are unassigned, what was the payer breakdown? Were any Medicare cases left unassigned?
  • Is there an opportunity to level out case assignments, i.e., can work be distributed across team members to help cover caseload?
  • How many discharged cases are unassigned?

Navigation: On the Analytics home page left side panel, navigate to CDE One > CDI Operations > Encounter Monitoring.

Encounter Monitoring displays the following:
  • KPI Card: Encounters
  • Encounter Count by Payer Chart
  • Encounter Count by Unit Chart
  • Encounter Count by Status Chart
  • Encounter Count by CDS Chart

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets filtered in the charts.

For filters all pages, refer Filters on all Pages.

For View, refer View.

Encounter Monitoring

Encounter Monitoring - KPI Card

This KPI Card displays the total count of encounters based on the 'View' selected.

Example: If you select 'Concurrent Encounters' from the 'View' dropdown list, this card displays the total count of concurrent encounters.

Encounter Monitoring - Encounter Count by Payer Chart

  • This chart displays the count of encounters for each payer, depending upon the View selected.

    Example: When you select ‘Unassigned Encounters’ from the 'View' drop-down list, this chart displays the count of Unassigned encounters for each payer.

  • Encounters that do not have a payer assigned are displayed under the 'Unassigned Payer' category.

  • X-axis: Displays the Payer in the format Payer Description
  • Y-axis: Displays the count of encounters.
  • Sort: Sort by Payer with the highest count of encounters to the payer with the lowest count of encounters (descending order).

Encounter Monitoring - Encounter Count by Unit Chart

  • This chart displays the count of encounters for each Unit, depending upon the measure selected.

    Example: When you select ‘Unassigned Encounters’ from the 'View' drop-down list, this chart displays the count of Unassigned encounters for each Unit.

  • Encounters that do not have a Unit assigned are displayed under the 'Unassigned Unit' category.

  • X-axis: Displays the Unit.
  • Y-axis: Displays the count of encounters.
  • Sort: Sort by Unit with the highest count of encounters to the Unit with the lowest count of encounters (descending order).

Encounter Monitoring - Encounter Count by Status Chart

  • This chart displays the count of encounters for each Status, depending upon the View selected.

    Example: When you select the Unassigned from the 'View' drop-down list, this chart displays the count of Unassigned encounters for each Status.

  • Applicable statuses are displayed based on the selected measure. The following encounter statuses are applicable for each measure.
    View Encounter statuses
    Unassigned Encounters Discharged, Untouched
    Concurrent Encounters Untouched, Discharged, In Progress, Review Complete, Ready to Reconcile, Reconciliation Hold
    Non-Reconciled Encounters Review Complete, Ready to Reconcile, Reconciliation Hold
    Reconciled Encounters Reconciled-Impact, Reconciled No Impact, Auto-Reconciled
  • X-axis: Displays the Encounter Status.
  • Y-axis: Displays the count of encounters.
  • Sort: Sort by Status with the highest count of encounters to the Status with the lowest count of encounters (descending order).

Encounter Monitoring - Encounter Count by CDS Chart

  • This chart displays the count of encounters for each CDS, depending upon the View selected.

    Example: When you select ‘Unassigned Encounters’ from the 'View' drop-down list, this chart displays the count of Unassigned encounters for each CDS.

  • Encounters that do not have a CDS assigned are displayed under the 'Unassigned CDS' category.

  • X-axis: Displays the CDS in the format CDS Name (CDS ID)
  • Y-axis: Displays the count of encounters.
  • Sort: Sort by CDS with the highest count of encounters to the CDS with the lowest count of encounters (descending order).
Table 1. Report Slicers
Slicer Description
Admit Date
  • Displays the admit date range here. You can select required dates from the date lookup.
  • Example: 7/1/2020 - 7/31/2020
  • Default: Last 1 calendar months date range selected by default.
Discharge Date
  • Displays the discharge date range here. You can select required dates from the date lookup.

  • Example: 7/1/2020 - 7/31/2020
Reset
  • With the Reset button, you can reset the admit date range.
  • Tooltip: Use "Ctrl + Click" to reset the Admit Date and Discharge Date.
View
  • Displays the list of View - Unassigned Encounters, Concurrent Encounters, Non-Reconciled Encounters, Reconciled Encounters, All Encounters.
  • You can select only one View.
  • Default: Unassigned Encounters.
Facility
  • Displays a list of active and inactive facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple facilities
  • You can search the facility by Facility ID or Description.
  • Format: Facility Description (Facility ID)
  • Default: ALL
Payer
  • Displays a list of active and inactive payers associated with the facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple payers
  • "Unassigned Payer" displays data for the encounters that do not have a payer assigned.
  • You can search the Payer by Payer ID or Description.
  • Format: Payer Description (Payer ID)
  • Default: ALL

Drill Through feature

Encounters by Payer

Right click on a bar, select Drill through option, and then select Encounter Details. You get navigated to the ‘Encounter Details’ tab.

The ‘Encounter Details’ tab displays all the encounters for the selected Payer.

Encounters by Unit

Right click on a bar, select Drill through option, and then select Encounter Details. You get navigated to the ‘Encounter Details’ tab.

The ‘Encounter Details’ tab displays all the encounters for the selected Unit.

Encounters by Encounter Status

Right click on a bar, select Drill through option, and then select Encounter Details. You get navigated to the ‘Encounter Details’ tab.

The ‘Encounter Details’ tab displays all the encounters for the selected Encounter Status.

Note: The data displayed in the report is based on the "View" selected. For example, if you have selected the view as Unassigned encounters and encounter status filter as "In Progress", the system considers only Untouched and Discharged cases and there will not be any data for "In Progress" cases.

Encounters by CDS

Right click on a bar, select Drill through option, and then select Encounter Details. You get navigated to the ‘Encounter Details’ tab.

The ‘Encounter Details’ tab displays all the encounters that have the selected CDS assigned.
