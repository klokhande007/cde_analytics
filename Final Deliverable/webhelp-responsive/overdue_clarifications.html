Overdue Clarifications

Overview: Overdue Clarifications report enables the CDS Managers to analyze the summary and detailed information on the clarifications that are overdue and take immediate action.

The report provides answers to the following questions.
  • How many clarifications are overdue and require follow-up? Can the physician liaison help?
  • Which providers have the most overdue clarification instances?
  • How do I get a list of clarifications that are overdue?
  • Which CDSs have the most overdue clarifications?
  • What is the detail around overdue clarifications?

Navigation: On the Analytics home page left side panel, navigate to CDE One > CDI Operations > Overdue Clarifications.

Overdue Clarifications displays the following:
  • KPI Cards: Encounters with Overdue Clarifications, Overdue Clarifications, Overdue Instances
  • Overdue Clarification Instances by Provider Chart
  • Overdue Clarification Instances by CDS Chart
  • Overdue Clarification Instances Tabular Chart

On the top panel, you can select desired values for the slicers (refer Report Slicers) and the corresponding data gets filtered in the charts and table.

For filters all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Default Display: By default, the dashboard displays the data for the inpatient encounters admitted in the last one calendar month. Click the ‘Reset’ button to reset admit date and select the required date.

Overdue Clarifications

Overdue Clarifications - KPI Cards
Table 1. KPI Cards
KPI Card Description
Encounters with Overdue Clarifications The count of encounters that have at least one Clarification with the status as Overdue.
Overdue Clarifications

The count of clarifications with the status as Overdue.

An encounter can have more than one overdue clarification.
Overdue Instances

The count of ‘Pending’ instances of the Clarifications with the status as Overdue.

Overdue Clarifications - Overdue Clarification Instances by Provider Chart

  • This chart displays the count of overdue clarification instances for each Provider in the selected date range and filter criteria.
  • The provider is the Provider displayed in the ‘Responded By’ field in the Clarification UI.
  • X-axis: Displays the name of the Responded By provider in the format Dr. Smith, Dennis B, Jr., MD
  • Y-axis: Displays the count of the overdue clarification instances.
  • An overdue clarification can have more than one ‘pending’ instances.
  • Sort: Data in the chart is sorted by the provider with the highest count of Overdue clarification instances to the provider with the lowest count of Overdue Clarification instances (descending order).
  • Info Icon(Text on hover): Count of overdue clarification instances for each Provider.; Instance Response = Pending

Overdue Clarifications - Overdue Clarification Instances by CDS Chart

  • This chart displays the count of overdue clarification instances for each Initiating CDS in the selected date range and filter criteria.
  • Initiating CDS is the CDS who sent the clarification.
  • X-axis: Displays the name of the initiating CDS in the format CDS ID.
  • Y-axis: Displays the count of the overdue clarification instances.
  • Sort: Data in the chart is sorted by the CDS with the highest count of Overdue clarification instances to the CDS with the lowest count of Overdue Clarification instances (descending Order).
  • Info Icon(Text on hover): Count of overdue clarification instances for every initiating CDS. ; Instance Response = Pending

Overdue Clarifications - Overdue Clarification Instances

  • This tabular displays the clarification instance details.
  • Each row in this report corresponds to one clarification instance.
  • Total No. of Rows must be equal to the count of overdue clarification instances.
  • Sort: Data in the chart is sorted by Instance Sent Date (ascending order).
  • Info Icon(Text on hover): Data in this chart is for all overdue clarification instances. ; Encounter Status: Includes all encounter status except Review Not Needed.
