CDS/Coder Match

Overview: CDS/Coder Match report enables the CDS Managers/Team Lead to analyze each encounter reviewed by the CDS and if W-DRG or P-DRG matched the coding DRG to assess the accuracy of the CDS’s work.

Navigation: On the Analytics home page left side panel, navigate to CDE One > Staff Management > CDS/Coder Match.

CDS/Coder Match dashboard displays the following:
  • KPI Cards: Reconciled Encounters, Matched Encounters, Unmatched Encounters, CDS/Coder Match Rate
  • CDS/Coder Match Rate chart
  • CDS/Coder Match Summary chart
  • Match Detail tabular chart

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets filtered in the tabular chart.

For filters all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Default Display: By default, the dashboard displays the data for the inpatient encounters discharged in the last three calendar months. Click the ‘Reset’ button to reset the discharge date and select the required date.

CDS/Coder Match Logic
  • Compares Final Coded DRG to the Possible DRG (or Working DRG when Possible DRG is Blank)
  • Reconciliation Assessment DRG is NOT used to calculate the CDS/Coder Match.
  • For reconciled encounters without Possible DRG (i.e. Possible DRG is Blank), CDS/Coder Match is "Y" when Final Coded DRG matches with Working DRG.
  • For reconciled encounters with Possible DRG , CDS/Coder Match is "Y" when Final Coded DRG matches with Possible DRG.
Note: Applicable to all Reconciled cases (Reconciled-Impact, Reconciled-No Impact, Auto-Reconciled) and Grouper Types (MS-DRG and APR Groupers)
