Tracking Codes

Overview: Tracking Codes report enables the CDS Managers to analyze all the encounters that have a tracking code assigned and identify coding, quality, and provider education opportunities.

The report provides answers to the following questions.
  • Which encounters have a particular tracking code assigned?
  • Which tracking codes are assigned by the CDS?
  • Which tracking codes are assigned to an encounter?

Navigation: On the Analytics home page left side panel, navigate to CDE One > CDI Operations > Tracking Codes.

Tracking Codes displays the following:
  • KPI Card: Encounters with Tracking Codes
  • Encounter Volume by Tracking Code Chart
  • Encounter Volume by CDS Chart

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets filtered in charts.

For filters all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Default Display: By default, the dashboard displays the data for the inpatient encounters admitted in the last one calendar month. Click the ‘Reset’ button to reset both admit date and discharge date and select the required date. Right click on individual bar charts and select ‘Drill Through:Encounter Details’ to view encounter level information.

Tracking Codes

Tracking Codes - KPI Card
Table 1. KPI Card details
KPI Card Definition
Encounters with Tracking Codes The count of total encounters with at least one Tracking Code.
Note: The count of encounters displayed on the "Encounter Volume by Tracking Code" will not match with this count, as the chart shows one to many mappings, where a single encounter is mapped to more than one tracking code.

Tracking Codes - Encounter Volume by Tracking Code Chart

This chart displays the different tracking codes and the volume of encounters they are assigned to.

X- axis: Displays the Tracking Code ID along with the Tracking Code Descriptions. Format: Tracking Code ID -Tracking Code Descriptions.

Y- axis: Displays the Encounter count.

Info icon(text on hover): Count of encounters and different tracking codes assigned to them.

Tracking Codes - Encounter Volume by CDS Chart

This chart displays the volume of encounters by each tracking code for every CDS.

X-axis: Displays the CDS name in ascending order. (Format: CDS ID)

Y- axis: Displays the Encounter count.

Info icon(text on hover): Count of encounters by each tracking code for every CDS.

Note: These are not one to one mappings, and there is a possibility that one encounter has more than one tracking code. A single encounter can have a maximum of four Tracking codes.

Report Chart Drill-through Functionality

Tracking Codes Encounter Volume by CDS

Table 3. Report Slicers
Slicer Description
Admit Date
  • Displays the admit date range here. You can select required dates from the date lookup.
  • Example: 7/1/2020 - 7/31/2020
  • Default: Last 1 calendar months date range selected by default.
Discharge Date
  • Displays the discharge date range here. You can select required dates from the date lookup.

  • Example: 7/1/2020 - 7/31/2020
Reset
  • With the Reset button, you can reset the admit date range.
  • Tooltip: Use "Ctrl + Click" to reset the Admit Date and Discharge Date.
Tracking Code
  • Displays a list of active and inactive Tracking Codes along with descriptions.

    Format: Tracking Code ID - Tracking Code Descriptions.

  • This filter allows you to view and select multiple Tracking Codes from the drop down.
  • This filter should not display the inactive Tracking Codes that are not assigned to any encounter.
Facility
  • Displays a list of active and inactive facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple facilities
  • You can search the facility by Facility ID or Description.
  • Format: Facility Description (Facility ID)
  • Default: ALL
Payer
  • Displays a list of active and inactive payers associated with the facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple payers
  • "Unassigned Payer" displays data for the encounters that do not have a payer assigned.
  • You can search the Payer by Payer ID or Description.
  • Format: Payer Description (Payer ID)
  • Default: ALL
CDS
  • Displays a list of active and inactive CDSs associated with the facilities you have access to.
  • The CDS Filter in all the reports displays the users with at-least CDS or CDS Manager role assigned.
  • "Unassigned CDS" displays data for the encounters that do not have a CDS assigned.
  • Format: Last Name, First Name MI (CDS ID)
  • Default: ALL
