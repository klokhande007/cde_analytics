CDS Impact

Overview: CDS Impact dashboard enables the CDS Managers to view the trend of various financial metrics over time and analyze the financial impact for the CDSs.

You can view the individual and detailed performance of the CDSs to analyze their activity. Under the CDS column, select a CDS, right-click, and select Drill through > CDS Performance, the system navigates to the CDS Performance tab. This tab displays all the data for the selected CDS.
Note:
When the CDS Performance tab appears, from the title of the tab, click CTRL + to go back to the previous tab (from the tab you have used the Drill through) command). In this case, it is CDS Impact.

Navigation: On the Analytics home page left side panel, navigate to CDE One > Staff Management > CDS Impact

CDS Impact displays the CDS Impact chart.

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets filtered in the tabular chart.

For filters all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Default Display: By default, the dashboard displays the data for the inpatient encounters discharged in the last nine calendar months. Click the ‘Reset’ button to reset the discharge date and select the required date. Total column displays the row-level grand totals for the selected period. For the percentage metrics, this column displays the overall percentage for the selected period.

Table 1. CDS Impact Chart Metrics
Metric Description Formula/Calculation Exclusion Criteria (if any) Tooltip
CMI Before CDI Displays the CMI Before CDI value for each CDS. CMI Before CDI = The sum of Final Coded DRG WTs of all Auto-Reconciled, Reconciled- No Impact, and Reconciled Impact (Impact Type: SOI/ROM Impact) encounters and Reconciliation assessment DRG WT (or Working DRG WT when Reconciliation Assessment Info is not available) of all Reconciled Impact (Impact Type: Both, Financial, (Unassigned (DD< 11/14/2019)) encounters that meet the report criteria / Total count of reconciled cases (Auto-Reconciled, Reconciled-Impact, Reconciled-No Impact) for the selected report criteria. This value is represented with 4 decimals. N/A
CMI After CDI Displays the CMI Shift value for each CDS. CMI After CDI = The sum of Final Coded DRG WTs of all Auto-Reconciled, Reconciled- No Impact, and Reconciled Impact (Impact Type: SOI/ROM Impact, Both, Financial, (Unassigned (DD< 11/14/2019)) encounters that meet the report criteria / Total count of reconciled cases (Auto-Reconciled, Reconciled-Impact, Reconciled-No Impact) for the selected report criteria. This value is represented with 4 decimals.
CMI Shift Displays the CMI Shift value for each CDS. CMI Shift = The sum of CMI Shift (CMI After CDI - CMI Before CDI) for the Reconciled-Impact (Impact Type: Financial, Both, Unassigned Impact for Discharge Date < 11/14/2019) encounters that meet the report criteria. (Rounded 4 decimal places E.g., 2.1868)/ Total count of reconciled cases (Auto-Reconciled, Reconciled-Impact, Reconciled-No Impact) for the selected report criteria. This value is represented with 4 decimals.
CMI Shift (%) Displays the CMI Shift % value for each CDS. CMI Shift % = (The sum of CMI Shift (CMI After CDI - CMI Before CDI) for the Reconciled-Impact (Impact Type: Financial, Both, Unassigned Impact for Discharge Date < 11/14/2019) encounters that meet the report criteria) / (CMI Before CDI for Auto Reconciled, Reconciled-No Impact, , Reconciled-Impact (SOI/ROM Impact) Reconciled-Impact (Impact Type: Financial, Both, Unassigned Impact for Discharge Date < 11/14/2019) encounters that meet the report criteria). X 100. Rounded to 2 decimal places. E.g. 2.74%.
% Cases with SOI Impact

This metric displays the percentage of reconciled cases that had an upward shift in the SOI (SOI After > SOI Before) due to CDS' work on the encounters.

Note:

Advanced SOI/ROM Values are used if Primary SOI/ROM values are blank.

 % cases with SOI Impact= The total count of Reconciled encounters(Auto Reconciled, Reconciled-Impact and Reconciled-No Impact) for which the SOI After is greater than SOI Before./ Total count of Reconciled encounters(Auto Reconciled, Reconciled-Impact and Reconciled-No Impact)
% Cases with ROM Impact This metric displays the percentage of reconciled cases that had an upward shift in the ROM(ROM After > ROM Before) due to CDS' work on the encounters. %  cases with ROM Impact= The total count of Reconciled encounters(Auto Reconciled, Reconciled-Impact and Reconciled-No Impact)  for which the ROM After is greater than ROM Before./ Total count of Reconciled encounters(Auto Reconciled, Reconciled-Impact and Reconciled-No Impact)
Table 2. Report Slicers
Slicer Description
Discharge Date
  • Displays the discharge date range here. You can select required dates from the date lookup.

  • Example: 7/1/2020 - 7/31/2020

  • Default: Last 9 calendar months date range
Reset
  • With the Reset button, you can reset the discharge date range.
  • Tooltip: Use "Ctrl + Click" to reset the Discharge Date.
Facility
  • Displays a list of active and inactive facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple facilities
  • You can search the facility by Facility ID or Description.
  • Format: Facility Description (Facility ID)
  • Default: ALL
Payer
  • Displays a list of active and inactive payers associated with the facilities that you have access to. The list is sorted alphabetically.
  • You can select multiple payers
  • "Unassigned Payer" displays data for the encounters that do not have a payer assigned.
  • You can search the Payer by Payer ID or Description.
  • Format: Payer Description (Payer ID)
  • Default: ALL
CDS
  • Displays a list of active and inactive CDSs associated with the facilities you have access to.
  • The CDS Filter in all the reports displays the users with at-least CDS or CDS Manager role assigned.
  • "Unassigned CDS" displays data for the encounters that do not have a CDS assigned.
  • Format: Last Name, First Name MI (CDS ID)
  • Default: ALL
