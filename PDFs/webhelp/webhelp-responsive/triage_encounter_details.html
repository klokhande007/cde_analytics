Encounter and Suggestion Details

Overview: The Encounter and Suggestion Details dashboard provides encounters details to the CDI Manager, displaying information on their assigned rank, AI suggestions, CDS activity, and suggested DRG. This report helps to analyze the data and identify opportunities for improvement.

By default, the report displays data for last 1 calendar month based on the admission date.

Navigation: On the Analytics home page left side panel, navigate to CDE One > Triage and CDS Assistant Usage > Encounter and Suggestion Details.

The Encounter and Suggestion Details report displays the following:
  • KPI Card: Total Suggestions, Total Encounters
  • Encounter Details chart
  • Suggestion Details chart
Note:

  • Encounters that have been given a rank in CDE Triage are called ranked encounters. The current rank of an encounter may be blank.

  • Encounters are ranked based on the rules that are defined for the prioritization within CDE Triage.

  • The following encounter types are considered:
    • Both ranked (standard and custom) and unranked encounters.
    • Highest rank of the encounter.
    • Count of all encounters that have been assigned a rank and have an initial review date/time.
    • All encounter statuses except 'Review Not Needed' are considered.

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets generated in the tabular chart.

For filters on all pages, refer Filters on all Pages.

For information about footnotes, refer Footnotes.

The total count of encounters with all statuses including Review Not Needed for the reporting period.
Note:
When the Admit Date is applied, then this card displays the count of admitted encounters with all statuses including Review Not Needed. When the Discharge Date is applied, then this card displays the count of discharged encounters with all statuses including Review Not Needed.

Variance

Displays the shift in the WT at the encounter level.

Example: 1.0178.

Encounter Variance = Final Coded DRG WT - Reconciliation Assessment DRG WT (or Variance = Final Coded DRG WT - Working DRG WT when Discharge Date < 11/14/19 or Reconciliation Assessment DRG is blank). For encounters with blank Final DRG WT, Encounter Variance = Possible DRG WT - Working DRG WT ( If Possible DRG/WT is blank, Working DRG WT is considered. In this case Encounter Variance will always be zero).

Note:
  • Encounter Variance can be negative, positive, or zero.
  • Negative ‘Variance’ is displayed in the red font, E.g. 1.0178.
  • Encounter Variance is not calculated for the encounters that do not have an initial review date.
  • This value is rounded to 4 decimal places.

  • Format: Rank # (Rank 1, Rank 2 etc).

  • Default: ALL.
Current Rank

  • Displays all standard and custom ranks that can be assigned to the encounters. This slicer filters the data as per the current standard or custom rank assigned to an encounter.

  • User can select multiple Ranks.

  • Users can search the Rank number.

  • “Unranked” displays data for all encounters that are not ranked.

  • Format: Rank # (Rank 1, Rank 2 etc).

  • Default: ALL.
