Quality Alerts

Overview: Quality Alerts report enables the CDS Managers to accurately measure the CDS impact on PSI, PDI, NQI alerts and ensure timely action.

The report provides answers to the following questions:
  • How many alerts were generated for a specific PSI/PDI/NQI alert category?
  • How many alerts were resolved by adding a specific exclusion code?
  • How many quality clarifications were created? How many of those were agreed?
  • How many alerts were resolved as present on admission?
  • What are the details around alerts?

Navigation: On the Analytics home page left side panel, navigate to CDE One > CDI Operations > Quality Alerts.

Quality Alerts displays the following:
  • KPI Cards: Alerts, Resolved
  • Quality Alerts Summary Tabular Chart
  • Quality Alerts Detail Tabular Chart

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets generated in the tabular chart.

For filters all pages, refer Filters on all Pages.

For Group By Criteria, refer Group By Criteria.

For Footnotes, refer Footnotes.

Default Display: Default Display: By default, the dashboard displays the data for the inpatient encounters admitted in the last one calendar month. Click the ‘Reset’ button to reset both admit date and discharge date and select the required date.

Quality Alerts - Group By Criteria

You can group the information in the Quality Alerts Summary visual with the help of the following criteria.

Table 2. Group By Criteria
Group By Criteria Definition
Encounter CDS
  • Summarizes the quality alert information by CDS.
  • ‘Value/Group By’ column displays the name of the CDS assigned to the encounters with quality alerts (PSI/PDI/NQI).

    Example: Jones,Mary ; Earl, Tabitha

  • 'Value/Group By' column displays the list of the CDS in alphabetical order sorted by their last name.
  • Default option selected when you open the report
Alert Category
  • Summarizes the quality alert information by all the three Alert Categories (PSI, PDI, NQI).
  • 'Value/Group By' column displays the three Alert Categories in alphabetical order (NQI Topic, PDI Topic, PSI Topic).
PSI Topic
  • Summarizes the quality alert information by PSI Topics.
  • ‘Value/Group By’ column displays the ID and Description of the PSI Topics.

    Example: PSI-13 Postoperative Sepsis

  • 'Value/Group By' column displays the list of the PSI Topics in ascending order.
PDI Topic
  • Summarizes the quality alert information by PDI Topics.
  • ‘Value/Group By’ column displays the ID and Description of the PDI Topics.

    Example: PDI-09 Postoperative Respiratory Failure Rate

  • 'Value/Group By' column will display the list of the PDI Topics in ascending order.
NQI Topic
  • Summarizes the quality alert information by NQI Topics.
  • ‘Value/Group By’ column displays the ID and Description of the NQI Topics.

    Example: NQI-3 Neonatal Blood Stream Infection

  • 'Value/Group By' column displays the list of the NQI Topics in ascending order.
Admit Month
  • Provides a monthly summary of the quality alert information for the selected admit date range.

  • 'Value/Group By' column displays the month and year of the Admit Date selected in the below format:

    YYYY-MM(Mon) ,

    Example: 2020-05(May)

  • Data is sorted and displayed by Admit Date in ascending order. Click the column header in Quality Alerts Summary -Group by/Value column to sort it in descending order.

  • Example: If you select the Admit Date filter from 23rd Dec, 2019 to 5th Apr, 2020 the 'Value/Group By' column displays the information as below:

    2019-12(Dec) → (Quality alerts generated from 23rd Dec, 2019- 31st Dec, 2019 get displayed)

    2020-01(Jan)

    2020-02(Feb)

    2020-03(Mar)

    2020-04(Apr) → (Quality Alerts generated from 1st Apr, 2020- 5th Apr, 2020 should be displayed)

Admit Week

  • Provides a weekly summary of the quality alert information for the selected admit date range.

  • 'Value/Group By' column displays the month and year selected in the below format:

    YYYY-MM-DD - YYYY-MM-DD (Example : 2020-04-26 - 2020-05-02)

  • The default start day of the week is considered to be Sunday.
Admit Quarter

  • Provides quarterly summary of the quality alert information for the selected admit date range.

    Format:YYYY-MM(Mon) - YYYY-MM(Mon), Example: 2020-05(Mar) - 2020-05(May)

  • Example: If the user selects the Discharge Date filter from 23rd Dec, 2019 to 5th Apr, 2020 the 'Value/Group By' column will display the information as below:

    2019-12(Dec) → (Quality alerts generated from 23rd Dec, 2019- 31st Dec, 2019 should be displayed)

    2020-01(Jan)

    2020-03(Mar)

  • Provides a weekly summary of the quality alert information for the selected discharge date range.

    Format: YYYY-MM-DD - YYYY-MM-DD (Example : 2020-04-26 - 2020-05-02)

  • Provides a quarterly summary of the quality alert information for the selected discharge date range.

The count of the total active Quality Alerts corresponding to the Group By parameter selected.

Example: If "Encounter CDS" group by option is selected, the total Active Quality Alerts for the individual CDS's get displayed.
No Exclusion Out of the total quality alerts generated, the count of resolved alerts for which CDS selects "No Exclusion - NQI Present"/ "No Exclusion - PDI Present"/ "No Exclusion - PSI Present" option from the Global Exclusion pop up based on the alert category to resolve the alert.
With POA

Out of the total quality alerts generated, the count of resolved alerts for which CDS selects "Present on Admission" option.

Note:
POA option can be used to resolve the alert can be used in two ways:
  • Adding the POA directly in the Review grid.
  • Selecting "Present on Admission" option from the Global Exclusion pop up
With Other Global Exclusions Out of the total alerts generated, the count of resolved alerts with for which CDS selects a ‘Global Exclusion’ (other than POA and No Exclusion(Present) ) criteria. Example: Non-Elective Admission, No OR Procedure, MDC 14 (Pregnancy) PDX
With Specific Exclusion Out of the total alerts generated, the count of resolved alerts with ‘Specific Exclusion’ criteria set by the CDS.
Clarifications Sent

Out of the total alerts generated, the count of resolved alerts for which quality clarifications were sent by the CDS.

Note:
  • CDS can send the clarifications to resolve the alerts in two ways:
    • Selecting the "Create new clarification on Code" option from the Add to drop down option in the "Specific Exclusions" pop up.
    • Selecting the "New Clarification" link directly from the Quality Alerts panel.
  • Only one clarification can be sent for each quality alert.
This does not include the count of alerts that are active.
Clarifications Responded Out of total quality clarifications sent, the count of quality clarifications responded by the providers. Active Alerts are not be considered. Only clarifications with a Final response will be counted.
%Responded

The percentage of the total quality clarifications responded by the provider.

%Responded = ( Quality Clarifications Responded/Quality Clarifications Sent)*100
Clarifications Agreed Out of total quality clarifications responded, the count of quality clarifications agreed by the providers.
%Agreed

The percentage of the total quality clarifications agreed by the provider.

%Agreed = ( Quality Clarifications Agreed/Quality Clarifications Responded)*100
Totals

Shows the total count for each of the columns in this table. Percentage is shown only for %Responded and %Agreed columns.

Example: Total count of alerts generated for a single individual CDS.

 %Responded column total = (Clarifications Responded column total)/(Clarifications Sent column total) *100% Agreed column total = (Clarifications Agreed column total)/(Clarifications Responded column total) *100
