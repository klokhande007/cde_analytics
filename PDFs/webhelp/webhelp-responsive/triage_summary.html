Triage Summary

Overview: The Triage Summary dashboard helps CDI Managers to export all usage and impact metrics for CDE Triage into a single file, which enables them to analyze the overall effectiveness and impact of encounter prioritization in one view. The dashboard also helps you group data by week, month, quarter, and year and analyze trends over different time periods and analyze them more easily.

By default, the report displays data for last 9 calendar months based on the admission date.

Navigation: On the Analytics home page left side panel, navigate to CDE Triage > Triage and CDS Assistant Usage > Triage Summary.

The Triage Summary dashboard displays the following:
  • KPI Cards: Encounters Ranked, Ranked Encounters Reviewed, Avg. Review TAT (Hours), Missed Top Ranked Reviews.
  • Triage Usage Summary metrics.
Note:
  • Encounters that have been assigned a rank in CDE Triage are considered as ranked encounters. The current rank of the encounter may be empty or not specified.
  • Encounters are ranked only (based on the rules) that are defined for the prioritization within CDE Triage and if any encounter that does not meet the criteria is defined as an unranked encounter.
  • The following encounter types are considered:
    • Both ranked (standard and custom) and unranked encounters.
    • Highest rank of the encounter.
    • Count of all encounters that have been assigned a rank and have an initial review date/time.
    • Ranked Encounters that were reviewed (have initial review date/time) but were later changed to "Review Not Needed" are excluded from this metric calculations.

On the top panel, you can select desired values for the slicers (refer Report Slicers ) and the corresponding data gets filtered in the charts.

For View By option, refer View By.

For filters on all pages, refer Filters on all Pages.

For Footnotes, refer Footnotes.

Triage Summary - KPI Cards
Table 1. KPI Cards
KPI Card Description
Encounters Ranked
Displays the count and percentage of encounters ranked for the selected time period.
Note:
Encounters that have ever been assigned a rank in CDE Triage are considered as Ranked Encounters. The current rank of the encounter may be ‘Blank’ as well.
Format: Encounters Ranked (% Encounters Ranked) 76 (77%).
  • Encounters Ranked: Count of encounters that were ranked (based on the rules) for the selected time frame.
  • % of Encounters Ranked: Out of the total (admissions / discharges), percentage of encounters that were ranked for the selected time frame.
    • % Encounters Ranked = (Count of encounters ranked/ Total Encounters (admissions / discharges) X100. This value is rounded to 0 decimal places.
Ranked Encounters Reviewed

Displays the count and percentage of ranked encounters reviewed for the selected time period.

Format: Ranked Encounters Reviewed (% Ranked Encounters Reviewed) 67 (89%).

  • Ranked Encounters Reviewed: Count of all ranked encounters (standard and custom) that have an initial review date/time assigned to it.
  • % Ranked Encounters Reviewed: (Count of ranked encounters reviewed /Count of encounters ranked) X100. This value is rounded to 0 decimal places.
Avg. Review TAT (Hours)

Displays the average Review TAT in hours for the selected time period, i.e. the number of hours between Review Complete Date/Time and when the Working Review was first created for the selected time period.

The 'Review TAT (Hours)' for each encounter can be calculated as the difference between the 'Review Complete' (Date/Time stamp) and the 'Initial Review' (Date/Time stamp).

Review TAT (Hours) for each encounter = ‘Review Complete’ (Date/Time stamp) – Initial Review (Date/Time stamp).

Avg. Review TAT (Hours) = Sum of Review TAT (Hours) of all the reviewed encounters / Encounters Reviewed.

Note:
'Review Complete' (Date/Time stamp) is recorded when the CDS selects the encounter status as 'Review Complete' and saves the encounter.
Missed Top Ranked Reviews

Displays the count of encounters with the highest rank as 1, 2, or 6 that were not reviewed by the CDS. These encounters do not have an initial review date/time. The encounters may or may not have a discharge date/time assigned.

Missed Top Rank Reviews = Count of top-ranked encounters (Rank 1, 2, and 6) that do not have an initial review, i.e. where the Initial Review Date/Time is either empty or not specified.

Note:
Only top ranked (standard highest rank 1, 2 and 6) encounters are considered.

Triage Usage Summary

This chart enables report users to analyze all usage metrics for CDE Triage in the same view for the selected 'View By' option.

Info Icon(Text on hover): Includes encounters other than 'Review Not Needed'. For data on encounters with specific ranks, use the 'Highest Rank’ slicer.

Missed Top Rank Reviews (Rank 1,2 and 6)

  • Codes can be accepted for ‘Documented Diagnoses and Procedures’ and ‘Documentation Opportunities Diagnoses’ categories and are available under the 'Accepted’ drop down.

Suggestions Clarified

Displays the count of suggested codes for which CDS generated a clarification.

Note:

  • Deleted, Retracted and Failed clarifications are excluded by default.

  • Clarification is only generated for the Suggested codes within Documented Diagnoses, Documentation Opportunity Diagnoses, Documented Procedures, i.e. clarifications are only created for codes with Suggestion Type = Suggested.

  • Any clarification that was unsent, retracted, deleted or failed is excluded.
% Suggestions Clarified

Out of the total suggestions that were presented on CDS Assistant, the percentage of codes for which CDS generated a clarification.

% Suggestions Clarified = (Total count of suggestions for which CDS generated a query / Total count of suggestions presented on CDS Assistant) X100. This value is rounded to 0 decimal places.

Note:

  • Deleted, Retracted and Failed clarifications are excluded by default.

  • Clarifications is only generated for the Suggested codes within Documented Diagnoses, Documentation Opportunity Diagnoses, Documented Procedures, i.e. clarifications are only created for codes with Suggestion Type = Suggested.

  • Any clarification that was unsent, retracted, deleted or failed is excluded.
